## P0

### 1. 首页实时噪音卡仍是“最近”语义

涉及文件：

- `entry/src/main/ets/components/HomeView.ets`
- 可能涉及 `entry/src/main/ets/pages/Index.ets`
- 可能涉及 `entry/src/main/ets/model/DecibelMeter.ets`

设计稿要求：

- 首页卡片标题为 `环境噪音 · 实时`，右上角为 `LIVE` 状态。
- 卡片显示实时 dB 值、迷你频谱、等级渐变条和当前位置指示点。
- 分贝仪工具行右侧显示实时值，例如 `47 dB`。

当前差距：

- 代码使用 `环境噪音 · 最近` 和 `GO`，读取的是最近历史最高或最近记录语义，不是实时测量。
- 迷你等级条没有当前值位置指示点。
- 分贝仪工具行右侧显示固定文案 `测量`，不是当前 dB。
- 角度仪、指南针工具行也没有显示设计稿里的即时摘要值，例如 `0.0°`、`NE 42°`。

建议任务拆分：

- 增加首页轻量实时读数来源，避免与正式记录测量互相抢占麦克风；如果平台不允许后台持续麦克风，至少把 UI 文案改回“最近”，并在任务结果中说明与设计稿的产品取舍。
- 首页卡片按设计稿补齐 `LIVE` 状态、等级条指示点、`47 dB` 样式的工具行值。
- 给首页工具行补齐角度仪和指南针的摘要值，无法读取时显示 `--` 或当前项目已有的占位策略。

验收标准：

- 首页首屏语义与设计稿一致：实时就是实时，最近就是最近，不混用。
- 亮色和暗色下卡片、指示点、工具行值都使用主题令牌或既有等级色。
- 不新增硬编码等待；涉及传感器异步更新时使用确定的状态流或可控定时器封装，测试只验证公开状态或渲染语义。

### 2. 分贝仪待测态没有实时读数

涉及文件：

- `entry/src/main/ets/pages/DecibelPage.ets`
- `entry/src/main/ets/model/DecibelMeter.ets`
- `entry/src/main/ets/model/RecordStore.ets`

设计稿要求：

- `分贝仪 · 待测` 页面仍显示当前环境值，例如 `47`、`dB(A) · FAST`、等级胶囊 `安静 · 适合休息`。
- 按钮文案是“开始测量”，含义是开始记录这一段噪音变化，而不是首次启动读数。
- `分贝仪 · 测量中` 才显示频谱、MIN / AVG / MAX 和停止按钮。

当前差距：

- 未开始时中央读数显示 `--`，指针固定归位到 30dB，等级胶囊不显示。
- 只有点击开始后才开始采集，待测态没有实时预览。

建议任务拆分：

- 将“实时预览”和“记录测量”拆开：进入页面后可读取实时值；点击“开始测量”重置统计并开始保存本次记录。
- 待测态保留设计稿中的实时读数、指针和等级胶囊，但不展示 MIN / AVG / MAX。
- 退出页面时统一释放麦克风资源；权限拒绝时沿用现有错误提示。

验收标准：

- 待测态、测量中态和停止后的状态边界清晰，不重复保存记录。
- 切换 dB(A)/dB(C)、FAST/SLOW 时同时影响预览和记录测量。
- 增加针对状态转换的测试，测试名使用英文，按 AAA 结构组织。

## P1

### 3. 历史页趋势图仍是柱状图，未实现设计稿曲线图

涉及文件：

- `entry/src/main/ets/components/HistoryView.ets`
- `entry/src/main/ets/model/RecordStore.ets`

设计稿要求：

- 趋势卡是折线 / 面积图。
- 有 90 / 60 / 30 y 轴标记、分区背景、虚线网格、最近 7 天 x 轴标签。
- 顶部展示区间均值和峰值信息，例如 `58 dB 均值`、`峰值 93`、峰值发生时间。

当前差距：

- 趋势卡仅渲染最近若干条均值柱状图。
- 没有 y 轴、x 轴、分区背景、折线面积填充和峰值点。
- 峰值旁只显示记录条数，没有峰值发生时间。

建议任务拆分：

- 基于当前筛选区间聚合趋势数据，至少支持今天、7 天、30 天。
- 使用 ArkUI Canvas 或 Shape 绘制折线 / 面积图，不要用脆弱 DOM 结构模拟复杂曲线。
- 补齐峰值发生时间显示；无数据时保留当前空态。

验收标准：

- 有数据时趋势卡视觉结构与设计稿一致，亮暗色都可读。
- 无数据、单条数据、多条数据都不崩溃。
- 测试覆盖聚合逻辑、峰值时间选择、空数据行为。

### 4. 手电筒亮度控制未按设计稿实现

涉及文件：

- `entry/src/main/ets/pages/FlashlightPage.ets`
- `entry/src/main/ets/model/TorchController.ets`

设计稿要求：

- 常亮模式下显示 `亮度` 和百分比，例如 `80%`，滑块可用。
- 频闪和 SOS 是模式切换按钮。

当前差距：

- 常亮模式显示固定 `100%`，滑块禁用。
- 滑块当前只在频闪模式下作为频率控制。
- 如果系统手电筒 API 不支持亮度，UI 需要有明确降级策略，不能表现成可调但实际无效。

建议任务拆分：

- 查阅项目内或 HarmonyOS 官方相机 / torch API 能力，确认是否支持 torch 强度。
- 若支持，扩展 `TorchController` 增加亮度设置；若不支持，按设计保留展示但禁用并给出非打扰式说明。
- 将频闪频率控制与常亮亮度控制拆成不同 UI 状态，避免一个 slider 承担两个含义。

验收标准：

- 常亮、频闪、SOS 三种模式状态互不污染。
- 亮度不可用时不会误导用户。
- 频闪频率测试不要使用 `sleep()`，使用可控时钟或确定的状态断言。

### 5. 计时器运行态缺少右侧“复位”按钮

涉及文件：

- `entry/src/main/ets/pages/TimerPage.ets`

设计稿要求：

- 控制区同时有 `计次`、中间停止按钮、右侧 `复位`。
- 运行态也能看到右侧复位入口。

当前差距：

- 当前左侧按钮在运行时是 `计次`，暂停后变成 `复位`。
- 右侧只是占位 `Column().width(60).height(60)`，没有真实按钮。

建议任务拆分：

- 补齐右侧圆形 `复位` 按钮，位置和尺寸对齐设计稿。
- 明确运行中点击复位的行为：建议停止并清零；如果产品不允许运行中复位，则按钮可置灰但仍保持设计位置。
- 保留中间按钮的开始 / 停止状态。

验收标准：

- 控制区三按钮稳定占位，不因运行状态产生布局跳动。
- 计次列表顺序和格式保持当前设计稿样式。
- 增加 timer 状态转换测试，不使用硬编码等待。

### 6. 指南针精度文案和状态与设计稿不一致

涉及文件：

- `entry/src/main/ets/pages/CompassPage.ets`
- `entry/src/main/ets/model/CompassSensor.ets`

设计稿要求：

- 状态卡右侧为 `精度 ±2°`。

当前差距：

- 当前显示 `东北 · ±5°` 这类方向加固定精度。

建议任务拆分：

- 如果传感器 API 能提供精度，接入真实精度；否则使用产品认可的固定文案，但要统一设计稿和实现。
- 保持方向中文信息时，可另放在不影响设计稿主状态的位置。

验收标准：

- 状态卡和设计稿主文案一致。
- 无传感器数据时错误态明确且不显示虚假的“已校正”。

## 横向验收要求

- 亮色 / 暗色都要对齐，不允许只改一套颜色。
- 所有新增 UI 测试必须查询稳定语义或底层状态，不验证像素位置和深层组件树。
- 异步、传感器、计时器、频闪相关测试禁止 `sleep()` 和硬编码超时。
- 测试方法名只用英文，例如 `test_timer_reset_while_running_clears_elapsed_time`。
- 修改完成后依次执行：格式化 / 静态检查、全量测试、已配置的集成测试或真机测试标记。
